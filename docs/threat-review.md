# MVP threat review

Review date: 2026-07-15

This review covers the source-installable MVP, the `support-bash-3-via-blesh` compatibility change, native Unix terminal qualification, and the hardened required/scheduled/protected-manual CI boundaries. The security objective is narrow: provider output must remain inert text until it has crossed the local framing, decoding, command-policy, syntax, and risk checks, and a shell must never accept a generated command without a deliberate user Enter. Qualification must add no screen-capture, identity, credential-forwarding, persistent key-capture, writable fork, or secret-bearing evidence boundary.

## Reviewed boundaries

| Area | Controls and review result |
| --- | --- |
| Shell framing | Protocol-2 requests and responses use fixed-order NUL framing with exact field counts, per-field and whole-frame bounds, protocol negotiation, fully buffered responses, and request-ID matching. Backend/version metadata is bounded and validated locally; cursors are UTF-8 byte offsets with editor-native restoration state retained separately. Adapters assign the validated replacement as editor text; they do not `eval`, source, or execute it. |
| Bash editor backend | Bash 4.0+ uses native Readline unless the exact tested ble.sh editor is attached. Bash 3.2 requires that pinned editor and fails before bindings when it is absent, detached, wrongly ordered, API-incomplete, conflicted, or version-incompatible. History, clipboard, terminal scraping, comment-and-reprompt, and simulated-keystroke capture paths remain prohibited. |
| Provider subprocess | Official CLIs are invoked directly with argument arrays and prompt stdin, never through a shell. Each attempt uses a new private empty directory, an allowlisted environment, bounded output, a deadline, a separate process group, descendant cleanup, and removal on every exit path. Result files are opened nonblocking with no symlink following and must be regular files. |
| Output parsing | Provider JSON must be bounded, valid UTF-8, contain no duplicate object keys, contain exactly one value, and match one strict local `ok` or `clarify` branch with no unknown result fields. Claude's outer envelope is independently checked for duplicate keys before its structured result is decoded. |
| Command policy | Input is limited to one line and 8 KiB. The AST policy accepts one simple command or pipeline and recursively checks command/process substitutions; it rejects lists, background jobs, compounds, functions, comments, heredocs, and here-strings. Bash or Zsh then parses the text in no-execution mode with startup files disabled. |
| Risk rules | Every top-level and nested stage is considered. `sudo`, `env`, `command`, and `builtin` normalization has no fixed wrapper-depth cutoff. Environment changes, privilege, and explicit executable paths raise a result to at least review. Opaque `env -S`, download-to-shell through pipelines or process substitution, recursive/privileged deletion, raw disks, destructive Git/database operations, and shutdown patterns are dangerous. Provider hints can only raise local risk. |
| Enter confirmation | Generation never executes the target. Safe/review results retain editor-native one-Enter acceptance. A dangerous generated result requires a warning Enter followed by a second Enter while the exact generated text is still present. Rewrites, undo, cancellation, stale IDs, accepted commands, and a buffer that differs at a guard check clear the armed state. Native Bash resets its private macro continuation before every rewrite; ble.sh uses the pinned advice API and delegates only ordinary or second-unchanged acceptance. |
| Config and setup | Config is strict, secret-free TOML at an absolute XDG-derived path. Reads are bounded, nonblocking, and regular-file-only. Writes use a private temporary file, `0600`, sync, and atomic rename. Setup only performs a bounded nonblocking read of a regular startup file and prints reversible guidance; it never executes or modifies the file. |
| Chord configuration and rendering | One shared parser accepts only a bounded Alt+printable-ASCII or unreserved Ctrl+letter chord. It canonicalizes before comparison, keeps rewrite/undo distinct, and derives fixed one/two-byte sequences plus hex-only ZLE/Readline forms. Canonical markers are emitted as byte-escaped shell literals; adversarial punctuation never becomes shell syntax. Invalid `config set` leaves the previous atomic file unchanged. |
| Diagnostics | User-visible errors use curated messages passed through terminal-control removal and UTF-8-safe byte bounds. Raw prompts, model output, provider stderr, credentials, arbitrary subprocess errors, buffers, generated commands, binding bodies, and arbitrary received key text are not rendered. The explicit key probe opens `/dev/tty` rather than stdin, bounds time and bytes, symbolically renders mismatches, wipes received copies, and restores raw mode on normal and emergency paths. Doctor accepts only allowlisted bounded adapter markers, and provider subprocess filtering excludes those markers and ble.sh variables. |
| tmux and SSH qualification | tmux tests use an empty config, unique `-L` server name, private `TMUX_TMPDIR`, cleared parent `TMUX`, and `env -i` inner shells; they never contact the default server or capture panes. SSH tests require an explicit bounded target and existing non-interactive known-host authentication, disable forwarding/local commands, stage only an ephemeral cross-built fake-provider bundle, validate the canonical cleanup path, clear the remote environment, and never install or create a credential. Runtime model context retains only the existing remote boolean. |
| CI permissions and action supply chain | Every workflow declares top-level `contents: read`, rejects `pull_request_target`, write permissions, mutable action refs, credential-persisting checkout, fork-accessible secrets, and `continue-on-error`. Third-party actions use exact commit SHAs with reviewable version comments; ShellCheck, actionlint, OpenSpec, provider CLIs, shell sources, and vulnerability tooling are exact-version or checksum/integrity locked. Dependabot proposes reviewable Go and Actions updates but ordinary jobs receive no write token. |
| Downloaded fixtures and caches | ble.sh root and `contrib` archives, Bash 3.2, scheduled Bash/Zsh sources, and static tools are accepted only after checked-in SHA-256 verification. ble.sh assembles the pinned gitlink content before building and publishes a manifest-bound script atomically. Cache acceptance rechecks schema, commits, archive/build/installer digests, versions, licenses, regular-file type, and executability; stale, partial, corrupt, symlink, wrong-version, or interrupted state is rejected or rebuilt. |
| Ephemeral loopback SSH | Only the Linux required job may create the job-owned daemon/account boundary. It uses a high loopback port, temporary host/client keys and direct known-host data, a private isolated home, and disables passwords, forwarding, agent/X11, tunnels, user environment, and user rc files. Cleanup validates the PID/config/account/home namespace before killing descendants and removes only generated state. External SSH instead requires pre-existing credentials and never receives arbitrary option, port, identity, or config input. |
| Evidence, artifacts, and coverage | The Go JSON auditor discards all raw test output and accepts only manifest case, allow-listed phase, state, duration, and semantic matrix fields. Prompts, commands, PTY/pane streams, arbitrary environment/history, SSH material, and provider credentials are adversarially tested as absent; unknown matrix keys and credential-shaped values fail closed. Summary files are private, atomic, and reject links/non-regular destinations. Failure summaries expire after seven days. Reproducible artifacts are workflow-internal for one day; executable coverage stays in private local/remote directories, rejects non-regular remote entries, and uploads only the bounded policy summary on failure. |
| Protected manual qualification | Authenticated provider and caller-owned SSH tests exist only under `workflow_dispatch`, a dedicated self-hosted label, and a protected environment. They use existing logins/known hosts, never reference repository secrets, and upload no artifacts. Provider output is reduced to name, bounded version, and pass/fail. Hosted/fake evidence cannot refresh named terminal or authenticated provider records. |

## Findings closed during review

| ID | Finding | Resolution and regression evidence |
| --- | --- | --- |
| TR-01 | Go's default JSON behavior accepts duplicate keys and invalid UTF-8 replacement. | Reject invalid UTF-8 and duplicate keys recursively before decoding; cover result and Claude-envelope ambiguity in provider tests. |
| TR-02 | A provider could replace its result with a symlink or FIFO between inspection and open. | Open once with `O_NOFOLLOW` and `O_NONBLOCK`, then inspect the descriptor and accept only a regular file; test symlink and FIFO rejection. |
| TR-03 | A FIFO at a startup or config path could block a read-only command indefinitely. | Open nonblocking, inspect the opened descriptor, bound reads, and reject non-regular files; test prompt rejection. |
| TR-04 | Inline assignments and `env` could change `PATH` while a known basename remained classified safe. | Treat leading assignments and every normalized `env` layer as review; add direct and deep-wrapper cases. |
| TR-05 | `sudo ls` could inherit the read-only safe classification. | Any normalized privileged execution is at least review; destructive privileged operations remain dangerous. |
| TR-06 | `curl ... > >(sh)` and its `wget` equivalent bypassed the pipeline-only download-to-shell rule. | Relate top-level downloads to nested process-substitution shells and classify both forms as dangerous. |
| TR-07 | Bash could leave its private macro continuation mapped to `accept-line` after an earlier safe command. | Reset the continuation to a no-op before and after every rewrite; a PTY regression presses the private key directly against a newly generated dangerous marker command. |
| TR-08 | `/tmp/ls` inherited the `ls` safe rule even though the explicit path could name arbitrary code. | Explicit executable paths are at least review, including commands embedded in `find -exec`. |
| TR-09 | Eight or more supported wrappers could hide a destructive inner command. | Normalize supported wrappers until no wrapper remains; a 32-layer regression must still find recursive deletion. |
| TR-10 | `env -S` can turn one opaque string into a hidden command. | Treat split-string forms as dangerous shell evaluation, including attached long-option syntax. |
| TR-11 | Hosted PTYs can deliver Ctrl+C to Bash as `SIGINT` even when the adapter also supports raw control-byte cancellation; stock Bash 3.2 can crash if that signal interrupts a timed `read` inside a ble.sh widget. | Use a scoped native trap or ble.sh `INT` hook, poll terminal bytes in a separate monitor process, restore terminal/signal state, and cover both delivery modes with repeated stock-Bash PTY tests. |
| TR-12 | Bash 3.2 `bind -x` callbacks cannot safely read or replace the active line. | Require the exact tested attached ble.sh edit-command contract; plain Bash 3.2 fails before any integration binding, with Zsh and modern Bash as alternatives. |
| TR-13 | Replacing one ble.sh key before discovering a later conflict could leave a partial integration. | Preflight `M-g`, `M-u`, and accept-line advice, refuse unsupported conflicts, and restore earlier keys if a later installation step fails. PTYs preserve existing bindings across every negative mode. |
| TR-14 | Detachment or loss of a pinned internal ble.sh API could leave stale acceptance behavior active. | Recheck attachment, version, keymap, advice, and required functions on every interactive action; detach clears risk state and requires explicit reinitialization instead of backend fallback. |
| TR-15 | Arbitrary environment or binding text could leak through new backend diagnostics. | Export only fixed status markers, validate them against short allowlists, render only curated messages/key names, and exclude all adapter/ble.sh variables from provider environments and prompts. |
| TR-16 | A descriptor outside `select(2)`'s fixed set, an EOF-ready descriptor, or an unexpected session panic could panic/spin after `/dev/tty` entered raw mode. | Reject out-of-range descriptors before raw mode and before every set operation, treat a zero-byte ready read as EOF, and install an emergency restore/close defer immediately after opening. `TestTTYReadBoundedReturnsEOFWithoutSpinning`, `TestProbeEmergencyGuardRestoresAfterUnexpectedSessionPanic`, and the real PTY signal/timeout matrix verify restoration. |
| TR-17 | Partial or mismatched raw key bytes had more than one in-memory copy and were not explicitly zeroed on every error path. | Centralize byte wiping, clear read scratch buffers after copying, wipe accumulated bytes on all failures, and wipe the comparison slice after every step. `TestProbeWipesReceivedBytesAfterComparisonAndReadFailure` retains test references and verifies zeroing; privacy tests still reject raw text/control rendering. |
| TR-18 | A developer running tests from inside tmux could pass the parent `TMUX` and socket directory into the harness, while an inner shell inherited unrelated host environment values. | Clear `TMUX`, force a test-private `TMUX_TMPDIR`, keep the unique `-L` name and empty config, and launch inner shells with `env -i` plus a fixed test environment. `TestTmuxHarnessUsesPrivateSocketEnvironmentAndCleanInnerShell` covers the isolation contract. |
| TR-19 | A hostile or malformed SSH target response could return a traversal-shaped cleanup path, and user SSH configuration could otherwise enable agent/X11 forwarding or a local command during qualification. | Accept only a bounded target token, require a canonical absolute `intent-sh-ssh.<suffix>` path with no dot components before staging or cleanup, restrict file assertions to canonical children, disable password prompts, all forwarding, agent/X11 forwarding, update-host behavior, and local commands, and redact target/temp paths from bounded terminal-safe errors. `TestSSHHarnessRejectsUnsafeCleanupPathsAndDisablesForwarding` covers adversarial targets, paths, and required options. |
| TR-20 | Raw `go test -json` output or caller-selected matrix values could turn prompts, commands, terminal bytes, environment/history, SSH keys, or provider tokens into retained CI artifacts. | Parse only state-bearing JSON fields, drop every output event, allow-list safe nested phases and semantic matrix values, bound input/events, write private JSON atomically, and reject symlink/non-regular destinations. Adversarial evidence tests cover every prohibited category. |
| TR-21 | Pull-request code could inherit a writable token, mutable action, persisted checkout credential, or privileged secret path. | Enforce read-only top-level permissions and immutable action refs across every workflow, disable checkout credential persistence, prohibit `pull_request_target`, secret-bearing fork workflows, write scopes, and continue-on-error, and regression-test all checked-in workflows. |
| TR-22 | A leaked SSH session or executable-coverage transfer could leave job processes, keys, remote state, or a crafted archive entry behind. | The loopback fixture owns one validated account/home/PID/config namespace and has an `always()` teardown; direct disconnect proves the remote provider tree is reaped; coverage download rejects any non-regular entry and a strict filename bound before tar; external cleanup remains restricted to the validated `intent-sh-ssh.*` root. |

## Residual limits

- The local classifier is a heuristic, not a shell sandbox. Aliases, functions, the pre-existing `PATH`, shell options, executable versions, unsupported wrappers/interpreters, filesystem state, and remote targets can change behavior. Unknown or obscured operations are at least review, but only recognized dangerous structures receive the two-Enter guard.
- Bash Readline and ZLE expose the current buffer at widget callbacks, not a durable edit-generation counter. If text is changed and then restored byte-for-byte between guard callbacks, the adapter can only observe the final exact value. It never accepts a different command under the old fingerprint, but it cannot prove that no transient edit occurred.
- The selected provider CLI is a trusted local executable running as the user. CLI isolation flags, the empty working directory, and the environment allowlist reduce accidental access; they are not an OS sandbox against a compromised provider binary. The CLI retains the home/config access needed for its official login and network service.
- A manual edit makes the line user-owned shell input. The adapter does not re-run the provider classifier on every keystroke, so ordinary shell acceptance applies after the generated text differs. The visible line and the user's Enter remain the final authority.
- ble.sh is separately maintained code executing inside the user's interactive shell and controlling its line editor. The integration pins one exact tested commit because guarded acceptance relies on a narrow internal advice API; users must assess that third-party trust boundary and manage updates independently.
- Bash 3.2 can be slower than modern Bash or Zsh, especially during editor startup or redraw. Performance does not justify a less reliable capture mechanism, and an unsupported or degraded backend remains fail-closed.
- Required CI still trusts GitHub-hosted runner images, GitHub's artifact service for the one-day internal distribution bundle, upstream HTTPS availability for empty caches, and the reviewed commits represented by pinned action SHAs. Pinning makes changes explicit; it does not prove an upstream artifact benign.
- Scheduled vulnerability results depend on the recorded external Go vulnerability database revision. A clean scan is time-scoped evidence, not proof that no vulnerability exists. musl PTY capacity is explicitly unavailable and foreign-architecture inspection is not runtime or terminal qualification.
- A protected self-hosted runner and environment are administrative controls outside repository code. Maintainers must restrict runner enrollment and environment approval, keep provider/SSH authentication user-owned, inspect the workspace after a manual run, and never broaden that workflow to pull-request triggers.

These limits are also reflected in the README trust-boundary and risk sections. They do not weaken the no-auto-execution invariant: neither the model, provider CLI, Go binary, nor adapter accepts the target command on the user's behalf.
